# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 신규 구축
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI, 관리자 UI, 백엔드 서버, 데이터베이스)
- **Complexity**: Complex (실시간 통신, 인증/세션 관리, 다수 기능)

---

## 1. 기술 스택 결정

| 영역 | 기술 |
|------|------|
| Backend | Java + Spring Boot |
| Frontend | React (TypeScript) |
| Database | PostgreSQL |
| 배포 환경 | Docker Compose (로컬/온프레미스) |
| 실시간 통신 (관리자) | Server-Sent Events (SSE) |
| 실시간 통신 (고객) | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 기반 |

---

## 2. 아키텍처 결정

- **단일 매장** 지원 (멀티테넌시 불필요)
- **관리자 계정**: 매장당 1개, 사전 생성 (회원가입 없음)
- **이미지 관리**: 외부 URL만 저장 (업로드 기능 없음)
- **모놀리식 구조**: 단일 Spring Boot 애플리케이션 + React SPA

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer)

#### FR-C01: 테이블 태블릿 자동 로그인
- 관리자가 초기 설정 1회 수행 (매장 ID, 테이블 번호, 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

#### FR-C02: 메뉴 조회
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시, 장바구니 비우기, 메뉴 화면으로 자동 리다이렉트
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 테이블 정보, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 현재 테이블 세션 주문만 표시
- 주문 상태 SSE 기반 실시간 업데이트

### 3.2 관리자용 기능 (Admin)

#### FR-A01: 매장 인증
- 매장 ID, 사용자명, 비밀번호 입력
- JWT 토큰 기반, 16시간 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 테이블별 카드 그리드 레이아웃
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

#### FR-A03: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호, 세션 생성)
- 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- 테이블 이용 완료 (세션 종료, 주문 이력 이동, 리셋)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지 URL)
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 4. 비기능 요구사항

### NFR-01: 성능
- 관리자 주문 모니터링: SSE로 2초 이내 신규 주문 표시
- 고객 주문 상태: SSE로 실시간 업데이트

### NFR-02: 보안
- JWT 토큰 기반 인증 (16시간 만료)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

### NFR-03: 데이터 무결성
- 주문 데이터 트랜잭션 처리
- 세션 기반 주문 그룹화

### NFR-04: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 레이아웃
- 명확한 시각적 피드백

## 5. 제외 범위

- 결제 처리 (PG사 연동, 영수증, 환불)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 업로드/리사이징
- 알림 시스템 (푸시, SMS, 이메일)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜미디어)
